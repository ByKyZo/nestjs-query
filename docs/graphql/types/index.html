Types

@nestjs-query/query-graphql provides a number of types. Most will be automatically generated for you when you create a resolver. However, you can use many of the types in your own code for custom endpoints.

The following examples are based on the following TodoItemDTO

import { FilterableField } from '@nestjs-query/query-graphql';
import { ObjectType, ID, GraphQLISODateTime } from 'type-graphql';
@ObjectType('TodoItem')
export class TodoItemDTO {
  @FilterableField(() => ID)
  id!: string;
  @FilterableField()
  title!: string;
  @FilterableField()
  completed!: boolean;
  @FilterableField(() => GraphQLISODateTime)
  created!: Date;
  @FilterableField(() => GraphQLISODateTime)
  updated!: Date;
}

#ArgsTypes

#QueryArgsType

Args type used in read endpoints to allow filtering, paging, and sorting.

The QueryArgsType will generate an ArgsType that will require the user to provide three arguments.

  • filter? - A filter that should be used to find records to update. See FilterType
  • paging? - Options to page of result. See CursorPagingType
  • sorting? - Optional array to allow sorting of results. See SortType.

#Usage

import { QueryArgsType } from '@nestjs-query/query-graphql';
import { ArgsType } from 'type-graphql';
import { TodoItemDTO } from './dto/todo-item.dto';
@ArgsType()
export class TodoItemQuery extends QueryArgsType(TodoItemDTO) {}

#CreateOneArgsType

Args type for createOne mutations.

NOTE Dont forget to annotate your DTO with @InputType() from type-graphql.

NOTE Your DTO should be one that you want to use for input. For example you may not want to let the end user to specify the created or updated fields so you would create a new DTO just for input.

#Usage

import { CreateOneArgsType } from '@nestjs-query/query-graphql';
import { ArgsType } from 'type-graphql';
import { TodoItemDTO } from './dto/todo-item.dto';
@ArgsType()
export class CreateOneTodoItemArgs extends CreateOneArgsType(TodoItemDTO) {}

#UpdateOneArgsType

Args type for updateOne mutations.

The UpdateOneArgsType will generate an ArgsType that will require the user to provide two arguments.

  • id - The id of the recrord to update
  • input - A record with fields to update.

NOTE Dont forget to annotate your DTO with @InputType() from type-graphql.

NOTE Your DTO should be one that you want to use for updates. For example you may not want to let the end user to update the created or updated fields so you would create a new DTO just for updates.

#Usage

import { UpdateOneArgsType } from '@nestjs-query/query-graphql';
import { ArgsType } from 'type-graphql';
import { TodoItemDTO } from './dto/todo-item.dto';
@ArgsType()
export class UpdateOneTodoItem extends UpdateOneArgsType(TodoItemDTO) {}

#UpdateManyArgsType

Args type for updateMany mutations.

The UpdateOneArgsType will generate an ArgsType that will require the user to provide two arguments.

  • filter - The filter that should be used to find records to update. See FilterType
  • input - The update to apply to each record found.

NOTE Dont forget to annotate your DTO with @InputType() from type-graphql.

NOTE Your DTO should be one that you want to use for input. For example you may not want to let the end user to specify the created or updated fields so you would create a new DTO just for input.

#Usage

In this example we use the read DTO for the FilterType, and a different update DTO.

import { CreateOneArgsType, FilterType } from '@nestjs-query/query-graphql';
import { ArgsType } from 'type-graphql';
import { TodoItemUpdateDTO } from './dto/todo-item-update.dto';
import { TodoItemDTO } from './dto/todo-item.dto';
@ArgsType()
export class CreateOneTodoItemArgs extends UpdateManyArgsType(
  FilterType(TodoItemDTO), // filter
  TodoItemUpdateDTO // update DTO
) {}

#DeleteOneArgsType

Args type for deleteOne mutations.

The DeleteOneArgsType will generate an ArgsType that will require the user to provide two arguments.

  • input - The id of the record to delete.

#Usage

import { DeleteOneArgsType } from '@nestjs-query/query-graphql';
import { ArgsType } from 'type-graphql';

#DeleteManyArgsType

Args type type for deleteMany mutations.

The DeleteManyArgsType will generate an ArgsType that will require the user to provide:

  • input - A filter used to find records to delete. See FilterType

#Usage

@ArgsType()

#InputTypes

#FilterType

GraphQL implementation of the @nestjs-query/core Filter type.

The FilterType is dynamically created based on the fields in the DTO annotated with @FilterableField.

Along with the dynamically create fields filter also has the following static fields:

  • and - Allows for joining multiple Filters with using an AND condition.

    For example, find all todo items that start with 'Foo' AND end in 'Bar'.

    todoItems(filter: {
      and: [
        {title: {like: "Foo%"}},
        {title: {like: "%Bar"}},
      ]
    }) {
      #...select your fields
    }

  • or - Allows for joining multiple Filters using an OR condition.

    For example, find all todo items that (are completed AND start with 'Foo') OR (are not completed and end in 'Bar').

    todoItems(filter: {
      or: [
        {title: {eq: "Foo"}, completed: {is: true}},
        {title: {eq: "Bar"}, completed: {is: false}},
      ]
    }) {
      #...select your fields
    }
  • Code
  • Schema
import { FilterType } from '@nestjs-query/query-graphql';
import { TodoItemDTO } from './dto/todo-item.dto';
export const TodoItemFilter = FilterType(TodoItemDTO)

#CursorPagingType

The CursorPagingType allows you to limit or page through a ConnectionType.

NOTE This type is automatically created when using QueryArgsType.

For more comprehensive examples take a look at the Paging Docs

#Forward Paging

  • first - The number of records to return. This is the equivalent to an limit.
  • after? - The cursor of the node from a connection to start from.
    • The most common use case is to set this to the endCursor from a ConnectionType.

#Backward Paging

  • before - The cursor of the node in a connection to start from.
    • The most common use case is to set this to the startCursor from a ConnectionType.
  • last - The number of records before the before parameter to return.
  • Code
  • Schema
import { CursorPagingType } from '@nestjs-query/query-graphql';
export const CursorPaging = CursorPagingType()

#SortType

The SortType allows you to sort results.

NOTE This type is automatically created when using QueryArgsType.

For more comprehensive examples take a look at the Sorting Docs

#Fields

  • field - The field to sort on. The is an ENUM of @FilterableFields defined in the DTO.
  • direction - The direction to sort the field ASC or DESC.
  • nulls? - On supported storage engines you can specify the null sort order NULLS_FIRST, NULLS_LAST
  • Code
  • Schema
import { CursorPagingType } from '@nestjs-query/query-graphql';
import { TodoItemDTO } from './dto/todo-item.dto';
export const TodoItemSort = SortType(TodoItemDTO)

#PartialInputType

Utility type to create a Partial from an ObjectType.

This is the type used by resolvers when a CreateDTOClass or UpdateDTOClass are not specified.

  • Schema
import { CursorPagingType } from '@nestjs-query/query-graphql';
import { InputType } from `type-graphql`;
import { TodoItemDTO } from './dto/todo-item.dto';
@InputType()
export class TodoItemPartialInput extends PartialInput(TodoItemDTO) { }
